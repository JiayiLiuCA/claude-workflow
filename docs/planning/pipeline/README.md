# 域详细参考

各子系统（域）的行为详细参考，一域一文件（如 `ocr.md`、`auth.md`、`billing.md`），由各 step 的 Close 阶段创建与维护；`PIPELINE.md` §7 保持索引。

## 收录标准——只写代码读不出来的

- 行为表与 edge case 处理约定（「输入 X 时系统怎么办」的完整枚举）
- 外部依赖的集成 / 改造点清单（上游升级时按此 manual merge）
- 实测结论：性能数字、benchmark、被推翻的假设、踩坑记录
- 给后续 step 的承诺与提示（「Step X 开始前需要先…」）

代码能回答的（函数签名、schema 定义、路由清单）**不要抄进来**——那些以实际代码为准，PIPELINE.md §5 只留一行索引。
